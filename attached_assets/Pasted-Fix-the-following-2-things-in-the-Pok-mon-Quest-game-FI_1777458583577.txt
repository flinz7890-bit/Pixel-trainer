─────────────────────────────────────────
FIX 1: LOCATION IMAGES NOT SHOWING
─────────────────────────────────────────
The Bulbapedia location images are not rendering in the town/route
scene banner. The current CSS night sky background is still showing
for all locations including Viridian City.

Find the zone scene component — the large rectangular banner that
shows the location art (currently a dark CSS night scene with a moon
and buildings). It is likely rendered using inline CSS background or
a canvas/div element.

Replace it with an <img> tag as the background of that container:

  const locationImg = LOCATION_IMAGES[currentZone?.name];

  // In the JSX, inside the scene container:
  {locationImg && (
    <img
      src={locationImg}
      alt={currentZone.name}
      onError={(e) => { e.target.style.display = 'none'; }}
      style={{
        position: 'absolute',
        top: 0, left: 0,
        width: '100%',
        height: '100%',
        objectFit: 'cover',
        imageRendering: 'pixelated',
        zIndex: 0,
        borderRadius: 'inherit',
      }}
    />
  )}

Make sure the scene container has position: relative so the image
sits behind the location name tag and other overlaid elements.

  Charmander: https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/4.png
